# 週次サマリー生成コマンド

## 目的
今週の作業ログをまとめて確認・振り返りを行い、マネージャ期待値に対する進捗フィードバックを提供する。

## ユーザの入力
#$ARGUMENTS

## 手順

1. 現在の日付と今週の範囲（月曜〜日曜）を確認する
2. `~/.claude/projects/-Users-s11308-CyberAgent-Dynalyst/memory/secretary/expectations.md` を読み、現在の四半期の期待値を把握する
3. `~/.claude/projects/-Users-s11308-CyberAgent-Dynalyst/memory/secretary/` から該当月のファイルを読み、今週分を抽出する
4. 今週の git log も各リポジトリからスキャンする
   - `~/CyberAgent/Dynalyst/` 配下の git リポジトリ
5. 今週の活動を簡潔にまとめる:
   - 完了した主要タスク
   - 進行中のタスク
   - 記録漏れがないかの確認（git log にはあるが /log にない活動）
6. **期待値フィードバック**を出す:
   - 今週の活動が期待値・重点テーマにどう紐づくか
   - 期待値に対して進捗が薄い領域があれば指摘
   - 来週に向けたアドバイス（「期待値Xに関する活動が少ないので意識すると良いかも」等）
7. 画面に表示する
8. 記録漏れがあれば追記するか確認する

## 出力フォーマット

```
### 今週のサマリー (MM/DD〜MM/DD)
- ...

### 期待値に対する進捗
| 期待値 | 今週の関連活動 | 状況 |
|--------|--------------|------|
| XXX    | ○○を実施     | 順調 / 要注意 / 未着手 |

### 来週へのアドバイス
- ...
```

## 注意事項
- 簡潔に表示する
- 記録漏れの検出が重要な役割
- フィードバックは建設的に。責めるトーンではなく、気づきを与える程度に
- 期待値ファイルが空や未設定の場合はフィードバック部分をスキップし、期待値の記入を促す
