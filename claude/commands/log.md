# 作業ログ記録コマンド

## 目的
日々の作業成果・達成事項を記録し、半期査定時に活用できるようにする。
査定フォームの項目（Dynalyst内のやったこと / Dynalyst外のやったこと / できたこと / アピールポイント / 課題）に対応できるよう、タグ付けして記録する。

## ユーザの入力
#$ARGUMENTS

## 手順

1. ユーザの入力を理解する。入力がない場合は、現在のセッションの会話内容から成果を要約して記録する。
2. 現在の日付を確認する（`date +%Y-%m-%d` を実行）
3. 現在のプロジェクト（ディレクトリ名）を確認する
4. 以下の情報を整理する(整理する上で深掘りが必要であればユーザに適宜質問する):
   - **スコープ**: `dynalyst`, `external`(`dynalyst`以外での活動)
   - **Project**: DynalystのProject名（例: `bidding-改善`, `infra-刷新`, `creative-最適化` 等）。Projectが不明な場合はユーザに確認する。Dynalyst外の場合は省略可。
   - **役割**: そのProjectでの役割（例: リード / メンバー / サポート）。省略時は `メンバー`
   - **カテゴリ**: 開発 / 調査 / ドキュメント / レビュー / ミーティング / 組織貢献 / その他
   - **内容**: 何をしたか（わかりやすく、必要に応じてリッチに）
   - **成果・インパクト**: 定量的・定性的な成果（あれば）
   - **アピール度**: `high` / `normal`（特に貢献が大きいものは `high`）
   - **課題・反省**: もっとこうすべきだったこと（あれば）
5. `~/.claude/projects/-Users-s11308-CyberAgent-Dynalyst/memory/secretary/` 配下の該当月ファイル（例: `2026-03.md`）に追記する
   - ファイルが存在しない場合は新規作成する
   - 該当日のセクションがなければ `## YYYY-MM-DD` を追加
   - 既に同日セクションがあれば、その下に追記
6. `~/.claude/projects/-Users-s11308-CyberAgent-Dynalyst/memory/secretary/expectations.md` を確認し、現在の四半期の期待値が設定されていれば、記録した内容が期待値のどれに紐づくか一言添える（例:「→ 期待値『XXX』に関連する活動です」）。期待値が未設定なら何も言わない。
7. 記録完了を報告する

## 記録フォーマット

```markdown
## YYYY-MM-DD
- **[スコープ][Project名][カテゴリ]** 内容の要約（リポジトリ名）
  - 役割: リード/メンバー/サポート
  - 成果: 定量的・定性的な成果
  - アピール: high（該当する場合のみ記載）
  - 課題: 反省点（該当する場合のみ記載）
```

## 例

```markdown
## 2026-01-15
- **[dynalyst][bidding-改善][開発]** conv-appsのAPI v2移行を完了（conv-apps）
  - 役割: リード
  - 成果: レスポンスタイム30%改善、エラー率0.1%以下に
  - アピール: high
- **[dynalyst][bidding-改善][調査]** bidログの短期ウィンドウ分析（conv-apps）
  - 成果: 最適なウィンドウサイズの特定
- **[dynalyst][infra-刷新][開発]** Terraform設定のリファクタリング（conv-formation）
  - 役割: サポート
  - 成果: デプロイ時間20%短縮
- **[external][組織貢献]** 新卒向けGo勉強会の講師を担当
  - 成果: 参加者10名、満足度高
  - アピール: high
```

## 注意事項
- 簡潔に書く。後で半期レビュー生成時にAIが要約するので、キーワードが残っていれば十分
- 技術的な試行錯誤の詳細についても簡潔に記載すると良い。
- 同じ日に複数回呼ばれた場合は追記する
