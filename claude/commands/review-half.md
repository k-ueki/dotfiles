# 半期レビュー生成コマンド

## 目的
蓄積された作業ログから、Dynalyst Dev査定フォームの各項目に対応するレビューを自動生成する。
良い査定評価を得られるよう入力を手助けすること。

## ユーザの入力
#$ARGUMENTS

引数の例:
- `2026H1` → 2026年上半期（1月〜6月）
- `2025H2` → 2025年下半期（7月〜12月）
- 引数なしの場合は現在の半期を対象とする

## 査定フォームの項目（出力はこの順で生成する）

1. **やったことをおしえてください(Dynalyst内)** - ログの `[dynalyst]` タグを集約
2. **やったことをおしえてください(Dynalyst外)** - ログの `[external]` タグを集約
3. **できたこと** - 定量/定性に分けて成果を整理
4. **アピールポイント** - `アピール: high` のものを中心に特に貢献が大きいものをピックアップ
5. **課題** - ログの課題・反省を集約 + 全体を振り返って改善点を提案

## 手順

1. 対象期間を決定する
   - H1 = 9月〜2月、H2 = 3月〜8月
   - 注意: 査定フォームの対象期間は半期の境目と1-2ヶ月ずれることがある（例: 2026/03の査定は2025/09〜2026/02が対象）。ユーザに対象期間を確認すること。
2. `~/.claude/projects/-Users-s11308-CyberAgent-Dynalyst/memory/secretary/` から対象月のファイルを全て読み込む
3. 対象期間の全プロジェクトの git log もスキャンする
   - `~/CyberAgent/Dynalyst/` 配下の各 git リポジトリ(conv-apps, conv-formation, conv-digdag-jobs2)を探索
   - `git log --oneline --after="開始日" --before="終了日" --author` でフィルタ
4. 作業ログとgit logを突合し、記録漏れがあればユーザに確認する
5. 査定フォームの各項目に対応するテキストを生成する
   - テキストはmarkdown形式で箇条書きで記載し、その下に詳細を記載する。
6. 結果を `~/.claude/projects/-Users-s11308-CyberAgent-Dynalyst/memory/secretary/summary-{期間}.md` に保存
7. 結果は `~/Documents/査定資料/` にも保存する
7. 画面にも表示する

## 出力フォーマット

```markdown
# Dynalyst Dev査定: YYYY/MM

対象期間: YYYY/MM〜YYYY/MM

---

## やったこと (Dynalyst内)

### Project: XXX（役割: リード）
- やったことの概要
- やったことの概要

### Project: YYY（役割: メンバー）
- やったことの概要
...

## やったこと (Dynalyst外)
- 組織貢献や外部活動の概要
...

## できたこと

### 定量
- 具体的な数値成果（レスポンスタイムN%改善、コストN%削減、etc.）
...

### 定性
- 技術的な改善、チームへの貢献、ナレッジ共有 etc.
...

## アピールポイント
- 特に貢献が大きかったこと（1-3つに絞る）
...

## 課題
- もっとこうすべきだったこと、改善点
...

---

## 参考: 数値で見る活動
- コミット数: N
- 対象リポジトリ: xxx, yyy
```

## 注意事項
- 必要な情報があればユーザに確認すること。
- 全ての処理においてultrathinkでしっかりと考えて作業を行うこと
- 謙虚すぎず、かといって誇張もしない、事実ベースの記述を心がける
- 「やったこと」だけでなく「その結果どうなったか」を意識する
- 箇条書きで簡潔に。フォームにコピペしやすい形式にする
- 定量成果は具体的な数値を入れる。曖昧な表現（「大幅に改善」等）は避ける
